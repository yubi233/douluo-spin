## 执行顺序

1. 构建期迁移 `canonAdditions`、`catalog`、`martialSoulCategories` 和 `martialSoulTiers`。旧文本只在迁移工具中读取；生成结果必须包含稳定 ASCII ID、实体元数据、条件和效果。
2. 将 `engine` 的资格和战力规则拆为事实注册、结构化元数据和命名 policy。每个 policy 必须是纯函数并有单测。
3. 将 `machine` 的队列调度拆为 Process Manager。每一个旧 task handler 必须对应一个明确的 `Task.process`、输入 signal 和事件链。
4. 将传记、模拟和编辑器作为事件日志消费者补齐，不让它们自行重解读剧情文本。
5. 为每个迁移簇补充 `tests/content`、`tests/core`、`tests/application` 和 Chrome P0 旅程；所有规则恢复后，提升内容版本并从 batch-01 重新开始 20 x 20 认证。
