## 最终认证协议

- 每次调用只处理一个 batch，不使用 shell 循环、并行批次或自动递增。
- 固定 20 批，每批 20 份，恰好 10 human + 10 beast。
- 每份传记全文阅读，并交叉核对 JSON audit、state、trace 和 ending。
- 每份独立 review 必须勾选 7 项清单并写结论。
- 每批只有在 20/20 自动审计、20/20 人工 passed、`no-defect-confirmed` 和 `npm run test:verify` 通过后才能关闭 gate。
- 任一代码或内容修复都会使整个 generation 失效，并从新 commit 的 batch-01 重启。
